Split compartment modelCompartment A is used for source volume group and latest-backup discovery. Compartment B is used for restore destination, isolated networking, VM launch, evidence, and cleanup.
State-driven cleanupThe state file is updated after each phase. If a run fails midway, cleanup uses recorded resources to terminate instances, detach disks, and delete restored volumes.
Validation boundaryThe built-in validation is control-plane only: restored group AVAILABLE, instance RUNNING, and block volume attachments ATTACHED.
